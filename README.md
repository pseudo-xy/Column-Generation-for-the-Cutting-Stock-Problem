# Column-Generation-for-the-Cutting-Stock-Problem
This project implements the Column Generation algorithm to solve the Cutting Stock Problem, a classic integer programming challenge.

This project implements the Column Generation algorithm to solve the Cutting Stock Problem, a classic integer programming challenge. A lumber company (Woodco) sells boards in three sizes — 3-ft, 5-ft, and 9-ft — with demands of 25, 20, and 15 pieces respectively. Raw boards are 17 feet long and must be cut into combinations of these sizes. The objective is to minimize the total number of raw boards used (and thus total waste).
The cutting stock problem has an exponential number of possible cutting patterns, making it impractical to enumerate all columns upfront. Column Generation addresses this by starting with a small set of initial patterns and iteratively adding only the most promising new ones.
The algorithm alternates between two subproblems. The Master LP is a continuous relaxation that decides how many times each current cutting pattern should be used to meet all demand constraints at minimum total boards. Solving the master LP yields dual variables (shadow prices) for each piece size, representing the marginal value of satisfying one additional unit of each demand.
The Pricing Subproblem is a knapsack integer program that uses these shadow prices to search for a new cutting pattern with positive reduced cost — one that, if added to the master, would improve the objective. The knapsack maximizes the weighted value of pieces cut from a single 17-ft board, where weights are the shadow prices. If the best possible new pattern has a reduced cost of zero or less, no further improvement is possible and the LP is optimal.
The loop continues until optimality is confirmed, printing shadow prices, active variables, and the best candidate pattern at each iteration. The final LP solution is then rounded up to obtain a feasible integer solution, and a demand satisfaction check verifies all piece requirements are met. Total boards used, wood cut, and waste are reported as the final output.
